サイト名称は **「ライフデンタル｜治療と費用のご案内」**（医院の正式ホームページとは役割が別の、コンサル提示・QR持ち帰り用の検討サイト）。

## 1. 基本ワークフロー

1. ページの**中身（文章・構成）は、claude.ai のプロジェクト側で確定**してから渡される。Cowork はゼロから内容を考えるのではなく、**渡された確定内容を、このサイトの作法でHTML化・反映する**のが役割。
2. 渡された内容で、(a) 新規ページを作る／(b) 既存ページを編集する／(c) トップページのリンクを更新する。
3. 反映は **コミット → プッシュ**。プッシュは GitHub Pages に即公開されるため、**プッシュ前に必ず確認を取る**（後述の「安全運用」）。

## 3. デザインシステム（厳守）

新規ページ・編集ページは、`inlay-crown.html` と**同じ見た目**にすること。独自の色・フォント・装飾を持ち込まない。

### 3-1. カラートークン（CSS変数。これ以外の色を増やさない）

```
--paper:#FFFFFF;     /* 背景・白 */
--soft:#F4F7F5;      /* 淡いセージ地（セクション交互の背景） */
--card:#EEF3F1;      /* カード地 */
--ink:#2E3A3F;       /* 本文 */
--ink-soft:#54636A;  /* やや薄い本文 */
--muted:#6E7B80;     /* 補足 */
--sage:#4F7A6C;      /* アクセント（深め・見出し/アイコン） */
--sage-2:#6E9C8E;    /* アクセント（明るめ） */
--line:#D8E0DD;      /* 罫線 */
```

### 3-2. 書体・トーン

- フォントは日本語ゴシック体（`--gothic` 変数の指定をそのまま使う）。
- 行間は広め（`line-height:1.8` 基準）、字間はわずかに開ける。
- スマホ縦表示が基本。`@media (min-width:760px)` でPC幅に対応する**レスポンシブ**にする。

### 3-3. 使い回す部品（`inlay-crown.html` 内のものを流用）

- セクション：`.section`（交互に `.section.soft` で淡い地）
- 見出し：`.eyebrow`（小見出し）＋ `h2.title`（大見出し）＋ `.lead`（リード文）
- アイコン：インライン SVG の `<symbol>` 定義を `<use>` で呼ぶ方式
- スクロール表示アニメ：`.reveal` クラス＋ページ末尾のスクリプト
- フッター：白地・上罫線・`life-logo.png`・医院名・「いい歯で、いい人生を。」

### 3-4. フッター（全ページ共通・固定）

```html
<footer style="background:#fff;border-top:1px solid var(--line);padding:36px 22px 40px;text-align:center;">
  <img src="life-logo.png" alt="ライフデンタルクリニック" style="width:112px;height:auto;margin:0 auto 13px;display:block;">
  <div class="fl">ライフデンタルクリニック</div>
  <div class="ft">いい歯で、いい人生を。</div>
</footer>
```

### 3-5. 検索非掲載（noindex）の徹底（厳守）

このサイトは**コンサルでの提示とQR持ち帰り**が用途で、検索からの流入は不要。検索に出ると医院HPと候補が並んで紛らわしいため、**全ページを検索非掲載にする**。

- すべてのHTMLの `<head>` 内に、次の1行を必ず入れる（位置は `<meta charset="UTF-8">` の直後）。

  ```html
  <meta name="robots" content="noindex">
  ```

- **`robots.txt` の `Disallow` は使わない。** クロールをブロックすると `noindex` が読まれず、かえってURLだけ検索に出てしまう。検索非掲載は `noindex` メタタグに一本化する。
- 注意：noindex は「検索に出さない」だけで、**URLを知っていれば誰でも閲覧できる**（鍵付き非公開ではない）。厳密な限定が必要になったら別途相談。

## 4. 新規治療ページを追加する手順

1. `inlay-crown.html` を複製し、命名規則（第2章）のファイル名にする。
2. 渡された確定内容に、本文・見出し・図を**差し替える**。デザイン部品・配色・フッターは変えない。
3. **`<head>` に `<meta name="robots" content="noindex">` が入っているか確認する**（3-5）。複製元から引き継がれるが、消えていないか必ず見る。
4. **`index.html` のトップを更新**する。該当治療のカードは現在「準備中」表示になっているので、これを**実リンクのカードに差し替える**。
   - 準備中カード（`<div class="hub-card soon">…<span class="soon-chip">準備中</span></div>`）を、
   - リンクカード（`<a class="hub-card" href="ファイル名.html">…<span class="hc-arrow">…矢印…</span></a>`）に変更する。
   - 見本は `index.html` の「インレー・クラウン治療」カード。
5. 反映（コミット→確認→プッシュ）。

## 5. 患者向けの言葉づかい（重要・厳守）

このサイトは患者さんが読むものです。**内部用語を絶対に表に出さない**こと。

### 5-1. 患者の前で使わない（サイトに書かない）語

- 通い方の段番号：**段1／段2／段3**
- 内部の通い方ラベル：**包括投資型／治療特化型／堅実予防型／最小コスト型**
- 組み立て方の型：**ビルドアップ型／ゴール起点型**
- 審美の内部段階：**審美高／審美中／審美低**
- 患者心理モード：**S1〜S5、困りごとモード〜根づきモード**
- 患者レーン：**初診レーン／部分治療レーン／包括治療レーン／メインテナンスレーン**
- CARE Cycle の記号・訳語：**C／A／R／E、関係形成／認識共有 など**
- 旧コード：**L0／L0-1〜L0-3、L0クレジット** などの記号
- 進行度コード **C1〜C4** も、患者向けページでは記号として出さない（必要なら言葉で説明する）

### 5-2. 患者向けに使う言い回し（こちらを使う）

- 通い方：**今を整える／気になる歯を残す／全体で守る**
- 組み立て方：**1本ずつ治す進め方／全体の設計図から決める進め方**
- 見た目：**見た目も整える方針／自然な見た目を保つ方針／機能を大切にする方針**
- 療法（使用可）：**対処療法／原因療法／重点治療**
  - ただし「対処療法」を使うときは必ず**「今の困りごとにこたえる役割」**を添え、下位・劣位に見せない。
- 名称：**いい歯への基本方針／知っておきたいお話／検査でわかったこと／○○のご案内／今日のお口のご報告** など、確定名称を使う。

### 5-3. 文章の語感（全ページ共通）

- 体験で語る（機能の羅列より「どうなるか」）。動詞由来の言葉（整える・残す・守る・治す・保つ）。
- **優劣・ランクを匂わせない**。選択肢は「目的の違い」で並べ、上下に見せない。
- 「リスク」「チェック」「ステップ」等の分析的カタカナ語を避ける（「メインテナンス」のみ例外）。
- **急かさない**。「すぐに全部決めなくてよい」「いつでも見直せる」という当院の姿勢を保つ。
- 保険否定・他院批判はしない。

## 6. 安全運用（Cowork へのお願い）

- **削除は原則しない**。特に `life-logo.png` と既存の各HTMLは、明示の指示がない限り消さない・リネームしない。
- **プッシュ（公開）前に必ず確認を取る**。何を・どのファイルに変更したかを要約して提示し、了承を得てから push する。
- コミットは**小さく・1目的ずつ**。コミットメッセージは日本語で、何をしたか分かる短文にする（例：「根管治療ページを追加し、トップのリンクを有効化」）。
- 公開後は、**トップから新ページへのリンクが開くか**、**フッターにロゴが表示されるか**を確認する。
- 大きな構成変更（カテゴリの増減・トップの作り替えなど）は、勝手に進めず**先に相談**する。
